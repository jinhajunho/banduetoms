# 반듯한시공 통합 관리 시스템

## 📋 프로젝트 개요
건설/시공 업체를 위한 **단일 페이지 애플리케이션(SPA)** 통합 관리 시스템으로, 견적서, 매출/매입, 경비지출, 실적, 업체정보 등을 효율적으로 관리할 수 있는 웹 기반 애플리케이션입니다.

## 🚀 개발 환경 실행 (1단계 적용 후)

```bash
npm install
npm run dev
```

- 브라우저에서 `http://localhost:3000` 접속
- `index.html`이 루트, CSS는 `src/styles/main.css`, JS는 `src/js/app.js`로 분리됨

## 🎯 접근 방법

### 메인 페이지 (통합 SPA)
- **파일명:** `index.html`
- **구조:** 단일 페이지 애플리케이션 (Single Page Application)
- **네비게이션:** 좌측 사이드바 메뉴를 통해 페이지 전환 (페이지 새로고침 없음)
- **URL:** `index.html#{페이지명}` (예: `index.html#estimate`, `index.html#contractors`)
- **기능:**
  - ✅ 견적서 관리
  - ✅ 경영실적관리 (개발 중)
  - ✅ 주간보고 (개발 중)
  - ✅ 미수금 (개발 중)
  - ✅ 업체정보관리 (개발 중)
  - ✅ 경비지출관리 (개발 중)
  - ✅ 사용자관리 (개발 중)

### 장점
✅ **페이지 새로고침 없음** - 빠른 화면 전환  
✅ **하나의 파일** - 관리 편의성 증가  
✅ **브라우저 뒤로가기/앞으로가기 지원** - URL 해시 기반 네비게이션  
✅ **통합 사이드바** - 일관된 UI/UX

## 🎯 주요 UI/UX 개선사항

### 저장/수정/삭제 버튼 설계 철학

**문제:**
1. 입력 필드가 `disabled`로 비활성화되어 입력 불가
2. 저장 기능 없이 입력만 하면 데이터 유실
3. 저장된 행을 수정할 수 없음
4. 실수로 삭제 버튼 클릭 시 데이터 손실

**해결 방안:**

| 상태 | 버튼 | 아이콘 | 색상 | 기능 |
|------|------|--------|------|------|
| **신규 행** | 저장 | 💾 fa-save | 파란색 | 확인 → 저장 → 수정/삭제로 변경 |
| **저장된 행** | 수정 | ✏️ fa-edit | 주황색 | 편집 모드 활성화 → 저장/취소로 변경 |
| **저장된 행** | 삭제 | 🗑 fa-trash | 빨간색 | 확인 → 삭제 + 경고 메시지 |
| **편집 중** | 저장 | 💾 fa-save | 파란색 | 확인 → 저장 → 수정/삭제로 변경 |
| **편집 중** | 취소 | ❌ fa-times | 회색 | 원본 데이터 복원 → 수정/삭제로 변경 |

**핵심 개선:**
1. **입력 필드 작동:** `disabled` → `readOnly` 변경으로 모든 필드 정상 작동
2. **3단계 상태 관리:**
   - 신규 (편집 가능, 저장만)
   - 저장됨 (읽기 전용, 수정/삭제)
   - 편집 중 (편집 가능, 저장/취소)
3. **확인 다이얼로그:**
   - 저장: "입력한 내용을 저장하시겠습니까?"
   - 수정 저장: "수정한 내용을 저장하시겠습니까?"
   - 삭제: "삭제된 데이터는 복구할 수 없습니다"
4. **시각적 피드백:**
   - 편집 가능: 배경 연한 회색 (`var(--gray-50)`)
   - 읽기 전용: 배경 흰색 (`white`)
   - 호버: 위로 이동 + 그림자
5. **원본 데이터 백업:** 수정 시작 시 `data-original` 속성에 저장 → 취소 시 복원
